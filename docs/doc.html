Overview

The goal of this project is to implement a system capable of recording, tracking, and analysis multiple sets of cardiac tissues remotely from within an incubator.

Workflow:

Setup

Simple setup guide. It is a docker image that can be deployed anywhere. For it to work properly with the recording hardware it is recommended that it be hosted on a remote VM.

These instruction are tested on an Ubuntu 18.04.3 (LTS) virtual machine hosted on Digital Ocean.

SSH into virtual machine:
              
                $ ssh root@ip-of-machine
Install docker:
  • Ubuntu
  • Arch
                $ snap install docker
                $ pacman -Sy docker 

                  $ systemctl start docker.service
Pull the latest image:
              
                $ docker pull jackallope/tracking_docker:latest
Deploy the image:
              
                $ docker run --rm -d -p 8080:8080 -v ~/shared:/app/static/uploads jackallope/tracking_docker:latest
Access:
              
                http://ip-of-machine:8080

Analysis

Analysis Dashboard:

Options Meaning Restrictions
Polynomial What degree polynomial should be used for the curve fitting. Less than window length
Window Length Number of coefficients used for smoothing. Odd
Greater than polynomial
Peak Sensitivity Threshold How sensitive it to finding peaks. Lower = More Sensitive. (0, 1)
Buffer Distance from peak before it starts looking for basepoints/frontpoints. If too large will miss basepoints/frontpoints
Minimum Distance Distance between peaks. Useful to ignore doublepeaks etc. If too large will miss peaks

Data

Videos and other analyzed data can be stored temporarily on the server while analysis is being carried out. It is recommended that when a set of tissues is finished analysis their data be moved to a more suitible long term location (ie hard drive or NAS).
